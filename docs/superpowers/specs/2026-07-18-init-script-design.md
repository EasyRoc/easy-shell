# 初始化脚本 设计文档

日期：2026-07-18
状态：待确认

## 背景

登录服务器后，经常要重复敲一些固定命令：进入项目目录、加载环境变量、查看服务状态等。希望连接成功后能自动执行一串预设命令。编辑连接弹窗中已预留"初始化"占位 tab，本次实现它。

## 目标

- 每个连接可配置一段初始化命令
- 打开终端连接成功后，自动逐行执行，效果和手动输入一致
- 可开关，不启用时行为与现在完全一致

## 需求范围

### 做

1. 连接配置新增初始化脚本：多行文本，每行一条命令
2. 编辑弹窗启用"初始化"tab：启用开关 + 多行文本编辑框
3. 终端连接成功（shell 就绪）后，自动把脚本逐行发送到终端，每行以回车结尾
4. 脚本内容在终端中可见（像用户自己敲的一样，方便确认执行了什么）

### 不做

- 判断命令执行结果（成功/失败）——只是模拟输入，不做结果分析
- 变量替换、条件逻辑等脚本编排能力
- 等待命令执行完再发下一条（逐行连续发送，由 shell 自己排队执行）
- 全局初始化脚本（只按连接单独配置）

## 技术设计

### 数据结构

`SSHConnection` 新增字段：

```ts
initScript?: {
  enabled: boolean
  content: string   // 多行文本，每行一条命令
} | null
```

### 执行流程（主进程 sshManager）

1. `connect()` 建立 shell 通道成功后，检查该连接是否配置了启用的初始化脚本
2. 若启用：延迟约 500ms（等远端 shell 提示符就绪），把脚本按行拆分，过滤空行，逐行写入通道，每行末尾加 `\n`
3. 之后终端进入正常交互状态，用户可继续手动输入

### 界面

编辑弹窗"初始化"tab：

- 启用开关
- 多行文本框（等宽字体，占位提示：`每行一条命令，例如：\ncd /opt/app\nsource venv/bin/activate`）
- 说明文字：连接成功后自动执行，仅模拟输入，不校验执行结果

### 错误处理

| 场景 | 处理 |
| --- | --- |
| 脚本为空或全是空行 | 不发送任何内容 |
| 连接在 500ms 内断开 | 发送前检查通道状态，已关闭则跳过 |
| 脚本含 Windows 换行符（\r\n） | 拆分时统一处理，避免多余回车 |

### 测试与验证

1. `npx tsc -b` 类型检查通过
2. 浏览器 mock 模式验证表单交互（tab 启用、保存回填）
3. 真实环境验证：
   - 配置 `echo hello` + `pwd` 等命令，连接后终端自动出现执行过程和结果
   - 不启用开关时，连接后无任何自动输入
   - 空脚本连接后无异常
