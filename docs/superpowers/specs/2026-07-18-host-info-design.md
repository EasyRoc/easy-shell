# 系统信息展示 设计文档

日期：2026-07-18
状态：待确认

## 背景

参考图的连接列表中有一列"信息"，直接显示每台服务器的系统（如 Ubuntu）、CPU 核数、内存、硬盘容量，不用登录就能对机器配置一目了然。目前 Easy Shell 的列表没有这一列。

## 目标

- 列表中直接看到每台服务器的系统、CPU、内存、硬盘信息
- 信息采集一次后缓存，列表加载不卡顿
- 信息过期或配置变更后可手动刷新

## 需求范围

### 做

1. 连接列表新增"信息"列，显示：系统发行版、CPU 核数、内存总量、系统盘容量（如 `Ubuntu · 8核 · 15G · 39G`）
2. 采集方式：
   - 连接成功打开终端后，自动采集一次并缓存
   - 列表每行提供"刷新信息"按钮，手动重新采集
3. 采集结果缓存到连接配置中，列表直接读缓存展示；从未采集过的显示 `-`
4. 采集命令面向 Linux 服务器（发行版、CPU、内存、硬盘）

### 不做

- 实时/定时自动刷新（只在连接后和手动点击时采集）
- CPU 型号、负载、各分区明细等详细指标（只展示概要四项）
- 非 Linux 系统（macOS/Windows 服务器）的适配（解析失败时显示 `-`）
- 终端页内的系统监控面板

## 技术设计

### 数据结构

`SSHConnection` 新增字段：

```ts
sysInfo?: {
  os: string        // 如 'Ubuntu 22.04'
  cpuCores: number  // 如 8
  memTotal: string  // 如 '15G'
  diskTotal: string // 如 '39G'
  collectedAt: number
} | null
```

### 采集流程（主进程 sshManager）

新增 `collectSystemInfo(connectionId)`：

1. 建立一条临时 SSH 连接（复用现有连接配置，含跳板/代理逻辑）
2. 依次执行命令并读取输出：
   - 系统：`cat /etc/os-release`（取 `PRETTY_NAME`）
   - CPU：`nproc`
   - 内存：`free -m`（取 Mem 行总量，换算为 G）
   - 硬盘：`df -h /`（取根分区总量）
3. 解析输出为结构化数据，写回连接配置的 `sysInfo` 字段
4. 断开临时连接，返回结果

若该连接已有打开的终端会话，则复用现有连接执行（`client.exec`），不再新建。

### 触发时机

- 自动：`ssh:connect` 成功后，主进程后台执行采集，完成后推送 `connections:changed` 让列表刷新（不阻塞终端打开）
- 手动：列表"信息"列内容旁放一个小刷新图标，点击调 `ssh:collectInfo`，采集期间图标转圈

### 界面

- 列表"信息"列：一行小字，格式 `系统 · N核 · 内存 · 硬盘`；未采集显示 `-`
- 刷新按钮：行内操作区或信息列末尾的小图标按钮
- 采集失败：该列显示 `-`，鼠标悬停提示失败原因

### 错误处理

| 场景 | 处理 |
| --- | --- |
| 连接失败（机器下线/密码改了） | 保留旧缓存，悬停提示"采集失败：原因" |
| 命令不存在或输出格式异常（非 Linux） | 对应项显示 `-`，其余正常项照常展示 |
| 采集过程中连接断开 | 中止采集，保留旧缓存 |
| 并发重复点击刷新 | 采集期间按钮禁用，防止重复触发 |

### 测试与验证

1. `npx tsc -b` 类型检查通过
2. 浏览器 mock 模式验证列表展示（mock 几组假数据）和刷新按钮交互
3. 真实环境验证：
   - 连接一台 Linux 服务器后，列表自动出现系统信息
   - 手动点刷新，信息更新
   - 服务器关机后点刷新，旧信息保留且有失败提示
   - 未采集过的新连接显示 `-`
