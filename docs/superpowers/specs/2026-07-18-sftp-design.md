# SFTP 文件管理 设计文档

日期：2026-07-18
状态：待确认

## 背景

日常运维经常需要在本地和服务器之间传文件：上传发布包、下载日志等。目前只能用命令行（scp/sftp 命令）操作，不够直观。希望在终端旁边开一个文件面板，直接浏览服务器目录、拖拽传文件。

## 目标

- 终端会话旁边可打开文件面板，浏览服务器文件
- 上传、下载、新建文件夹、删除、重命名等常用操作
- 复用已建立的 SSH 连接，不需要重新登录

## 需求范围

### 做

1. 终端页每个会话标签栏增加"文件"按钮，点击在终端右侧展开/收起文件面板
2. 文件面板功能：
   - 目录浏览：双击文件夹进入、面包屑路径返回上级、刷新
   - 文件列表：名称、大小、修改时间；文件夹排在前面
   - 新建文件夹、删除（文件/空文件夹）、重命名
3. 上传：拖拽本地文件/文件夹到面板，或点"上传"按钮选择文件；支持多文件
4. 下载：双击文件或选中后点"下载"，弹出系统保存对话框选择位置
5. 传输进度：面板底部显示当前传输的文件名和百分比，可取消
6. 初始目录为用户家目录（`~`）

### 不做

- 本地文件面板（只有远端一侧，本地靠系统文件管理器拖拽）
- 文件内容在线编辑、预览
- 断点续传
- 多文件并行传输（排队逐个传）
- 权限（chmod）修改界面

## 技术设计

### 主进程

新增 `sftpManager`：

- 基于现有 SSH 会话：调用该会话连接的 `client.sftp()` 获取 SFTP 通道，不新建连接
- 每个终端会话对应一个 SFTP 通道，会话断开时通道一并关闭
- 目录/文件操作：`readdir`、`stat`、`mkdir`、`unlink`、`rmdir`、`rename`
- 传输：`fastPut`（上传）、`fastGet`（下载），带进度回调
- 进度推送：通过 `webContents.send('sftp:progress:{sessionId}', {file, percent, done, error})` 实时推给渲染进程
- 取消传输：记录当前传输任务，收到取消指令后销毁通道并重建

新增 IPC：

| 通道 | 说明 |
| --- | --- |
| `sftp:open` / `sftp:close` | 打开/关闭某会话的 SFTP 通道 |
| `sftp:list` | 列目录（路径 → 文件数组） |
| `sftp:mkdir` / `sftp:delete` / `sftp:rename` | 目录与文件管理 |
| `sftp:upload` | 上传（本地路径数组 + 远端目录） |
| `sftp:download` | 下载（远端路径 + 本地保存路径，主进程弹保存对话框） |
| `sftp:cancel` | 取消当前传输 |

### 渲染进程

新增 `FilePanel.tsx` 组件：

- 位于终端页右侧，宽约 320px，可折叠
- 顶部：面包屑路径 + 刷新按钮 + 新建文件夹按钮 + 上传按钮
- 中部：文件列表（图标区分文件夹/文件，名称、大小、修改时间）
- 底部：传输进度条（仅传输时显示，含取消按钮）
- 拖拽上传：监听面板区域的 dragover/drop，从拖拽数据取本地文件路径
- 删除、重命名：行内操作按钮，删除前弹确认

### 数据流

1. 用户点"文件"→ 渲染进程调 `sftp:open` → 主进程在现有 SSH 连接上开 SFTP 通道
2. 面板加载家目录列表 → 后续浏览、管理操作都走对应 IPC
3. 上传/下载时主进程持续推送进度事件 → 面板底部进度条更新
4. 关闭面板或会话断开 → `sftp:close`，通道释放

### 错误处理

| 场景 | 处理 |
| --- | --- |
| 服务器不支持 SFTP 或通道打开失败 | 面板显示错误提示和重试按钮 |
| 目录无权限访问 | 列表区域显示"无权限访问该目录"，可返回上级 |
| 路径不存在（被外部删除） | 刷新时回到上级存在的目录 |
| 传输中断（网络断开） | 进度条变红显示失败原因，文件列表保持可操作 |
| 删除非空文件夹 | 提示"文件夹不为空，无法删除"（不做递归删除，防误删） |
| 同名文件冲突 | 上传前提示，选择覆盖或跳过 |

### 测试与验证

1. `npx tsc -b` 类型检查通过
2. 浏览器 mock 模式验证面板交互（mock 一套假目录数据）
3. 真实环境验证：
   - 打开面板浏览目录、进出文件夹、刷新
   - 上传小文件/大文件（看进度）、拖拽上传、多文件上传
   - 下载文件到本地，内容一致
   - 新建文件夹、重命名、删除
   - 传输中途取消；断开连接时面板表现正常

## 依赖变化

无（ssh2 自带 SFTP 能力）。
